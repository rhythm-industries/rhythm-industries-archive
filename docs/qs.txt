LIFECYCLE (closed loop > hoarding)
──────────────────────────────────

ACQUIRE  ->  USE/SERVICE/CALIBRATE  ->  SURPLUS?  ->  DECIDE  ->  RELEASE
  |             |                      |            |          |
  |             |                      |            |          └─ gift / re-home w/ provenance intact
  |             |                      |            |
  |             |                      |            └─ keep (belongs-to: instance vs collection)
  |             |                      |
  |             |                      └─ duplicates become intentional (not clutter)
  |             |
  └─ record origin + evidence (attribution always)


SYSTEM RULES (vectors / alignment)
─────────────────────────────────
1) Canon is simple. Evidence is layered. (link > embed)
2) Facts ≠ Notes ≠ Unknown. Unknown stays unknown.
3) Collection ≠ Asset ≠ Instance. Keep the split.
4) Medium honors object. (Tacoma->paper ; RIA->plain text/git)
5) Lifecycle completes. (acquire → decide → release is allowed)
6) Quiet by default. No dashboards. No noise. No performative tracking.

OUTPUTS (what the system *produces*)
───────────────────────────────────
- trust
- future recall
- provenance that survives time
- easy re-homing
- calm: "nothing behind the scenes"
